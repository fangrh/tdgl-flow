# Flux GitOps CD Design

**Goal:** Set up Flux-based pull GitOps so that GitHub Actions builds images, updates manifest tags in Git, and Flux auto-deploys the changes to the k3s cluster. Also adds Traefik Ingress for the data-viewer.

**Architecture:** GitHub Actions builds and pushes container images to GHCR, then commits the new SHA tag back into the K8s manifests in the same repo. Flux watches the repo, detects manifest changes, and reconciles the cluster state. Traefik (already on k3s) provides external access via Ingress.

**Tech Stack:** Flux v2 (flux bootstrap), kustomize, Traefik IngressRoute, GitHub Actions, GHCR

---

## 1. Flux Bootstrap

Install Flux on the k3s cluster using `flux bootstrap github`. This creates:

- `flux-system` namespace with source-controller and kustomize-controller
- A `GitRepository` source pointing at `github.com/fangrh/kubeflow-tdgl` branch `main`
- Deploy key configured for read access to the repo

**Bootstrap command:**
```bash
flux bootstrap github \
  --owner=fangrh \
  --repository=kubeflow-tdgl \
  --branch=main \
  --path=clusters/k3s \
  --personal
```

The `clusters/k3s/` directory will contain the Flux system resources (auto-generated by bootstrap) and the `Kustomization` resources that point at `infra/` and `services/`.

## 2. Kustomize Structure

Add `kustomization.yaml` files to compose manifests for Flux:

- `infra/kustomization.yaml` — groups `namespace.yaml` and `postgresql/k8s/*.yaml`
- `services/data-viewer/k8s/kustomization.yaml` — groups `deployment.yaml`, `service.yaml`, `secret.yaml`, `ingress.yaml`
- `services/generator/k8s/kustomization.yaml` — groups `job.yaml`

Flux `Kustomization` resources in `clusters/k3s/`:

- `tdgl-infra` — source: GitRepository, path: `infra/`, namespace: `flux-system`
- `tdgl-services` — source: GitRepository, path: `services/`, namespace: `flux-system`, dependsOn: `tdgl-infra`

## 3. CI Pipeline Change

**Current:** Actions builds image, tags `:latest` + `:sha`, pushes to GHCR. Manifests always reference `:latest`.

**New flow:**

1. Actions builds image, tags `:sha` only (no `:latest`)
2. Actions pushes to GHCR
3. On push to `main` only (not PRs), Actions updates the image tag in the manifest file:
   - `services/data-viewer/k8s/deployment.yaml`: changes `image: ghcr.io/fangrh/tdgl-data-viewer:<old-sha>` to `image: ghcr.io/fangrh/tdgl-data-viewer:<new-sha>`
   - `services/generator/k8s/job.yaml`: same pattern for generator
4. Actions commits the manifest change back to `main`
5. Flux detects the commit and reconciles → deploys the new version

**Implementation:** Use `sed` to replace the SHA tag in the manifest, then `git commit` and `git push`. Use `GITHUB_TOKEN` with `contents: write` permission for the commit-back.

**PR builds** still build and push images but do NOT update manifests. This prevents PRs from triggering deployments.

**Initial manifest state:** Before the first CI run, manifests use a known SHA tag (the current image digest). The CI will update it on the next push.

## 4. Traefik Ingress

Add `services/data-viewer/k8s/ingress.yaml` with a standard Kubernetes `Ingress` resource:

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: data-viewer
  namespace: tdgl
  annotations:
    traefik.ingress.kubernetes.io/router.entrypoints: web
spec:
  rules:
    - host: tdgl.local
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: data-viewer
                port:
                  number: 80
```

For local access, add `127.0.0.1 tdgl.local` to `/etc/hosts`. The viewer is accessible at `http://tdgl.local/viewer`.

## 5. Flux Kustomization Resources

File: `clusters/k3s/flux-system/kustomization.yaml` (auto-generated by bootstrap)

File: `clusters/k3s/flux-system/gotk-sync.yaml` (auto-generated, GitRepository source)

File: `clusters/k3s/tdgl-resources.yaml` — defines the two Kustomization resources:

```yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: tdgl-infra
  namespace: flux-system
spec:
  interval: 5m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./infra
  prune: true
---
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: tdgl-services
  namespace: flux-system
spec:
  interval: 5m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./services
  prune: true
  dependsOn:
    - name: tdgl-infra
```

**Top-level composition:**

`services/kustomization.yaml` — composes all service subdirectories:
```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - data-viewer/k8s/
  - generator/k8s/
```

This lets Flux's Kustomization with `path: ./services` find and apply all service manifests through a single entry point.
